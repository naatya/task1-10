#### **1.The CIA Triad: The Pillars of Security**



The CIA triad is the industry-standard model for designing security policies.



* ###### **Confidentiality:** Ensuring data is only accessible to authorized users.

&nbsp;	Example: In Banking, your account balance is encrypted so only you and bank staff can see it.



* ###### **Integrity:** Ensuring data is accurate and has not been tampered with.

&nbsp;	Example: In Social Media, if you post a status, integrity ensures that a hacker cannot change the text of your

##### **3. Attack Surfaces: Where the Doors Are**



The attack surface is the total sum of all possible points where an unauthorized user can try to enter or extract data.



* **Web/Mobile Apps:** Vulnerabilities in the code (like a login form).



* **APIs:** The "glue" connecting apps; if unsecured, they can leak massive databases.



* **Network:** Wi-Fi, routers, and firewalls.



* **Cloud Infrastructure:** Misconfigured S3 buckets or cloud permissions.

##### **6. The Data Flow \& 7. Where Attacks Happen**



In a typical architecture, data moves in a specific path. Security must be applied at every "hop.



1. **"User-->Application:** The user enters a password.
 Attack: Phishing or Keylogging (stealing the password at the source).
   
2. **Application-->Server:** The app sends the data over the internet.
    Attack: Man-in-the-Middle (MitM) (intercepting data while in transit).
   
3. **Server-->Database:** The server asks the database to verify the user.
    Attack: SQL Injection (tricking the database into dumping all user records).
   
4. **Database Storage:** Data sits on a disk.

   Attack: Data Breach (stealing the physical or virtual storage drives).
